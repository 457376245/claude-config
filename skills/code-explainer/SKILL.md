## What I do

- Explain code behavior step-by-step from entrypoint to side effects
- Build and walk through the full call chain and execution flow
- Track how data structures change across functions and layers
- Analyze edge cases, error handling, and non-happy paths
- Highlight concurrency, performance, and resource-management pitfalls
- Summarize risks, assumptions, and concrete improvement suggestions

## When to use me

Use this skill when you need a **deep, execution-level understanding** of existing code, such as:

- Understanding *what the code really does* and *why it's written this way*
- Tracing a request or event through multiple layers (controller → service → DAO, handler → pipeline, etc.)
- Reviewing unfamiliar or critical code paths before modification
- Debugging production issues, race conditions, or subtle bugs
- Onboarding new team members to complex systems

If key context is missing (target scope, entrypoint, or runtime assumptions), I will ask **1–3 precise clarifying questions** before proceeding.
