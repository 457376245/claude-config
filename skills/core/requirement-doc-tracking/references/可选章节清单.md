# 可选章节清单

默认只维护一个需求实施追踪文档。只有当需求实际涉及对应风险或交付内容时，才把以下章节追加到主文档中；不要为了完整性创建空章节。

## 接口变更

适用场景：新增、删除或修改 API、路由、请求参数、响应字段、错误码、权限要求、幂等规则。

建议记录：

- 接口名称、URL、Method。
- 请求参数、响应参数、错误码。
- 权限与鉴权规则。
- 幂等、兼容性和调用方影响。
- 示例请求与响应。

## 数据库变更

适用场景：新增或修改表、字段、索引、数据迁移、初始化数据、兼容老数据。

建议记录：

- 表名、字段、类型、默认值、是否允许为空。
- 索引设计和原因。
- 老数据处理方式。
- DDL / DML 执行顺序。
- 回滚或补偿思路。

## 上线与回滚

适用场景：涉及发布顺序、配置开关、数据脚本、灰度、缓存、MQ、定时任务、外部依赖或生产风险。

建议记录：

- 发布服务和顺序。
- 配置或脚本变更。
- 上线后验证步骤。
- 触发回滚的条件。
- 应用、配置、数据、缓存或消息的恢复方式。

## 测试用例

适用场景：业务规则较多，或普通验证记录不足以支撑验收标准。

建议记录：

- 正常场景。
- 异常场景。
- 边界场景。
- 权限场景。
- 兼容场景。
- 回归范围。

## 阶段性交接

适用场景：需求暂停、阻塞、换 agent 或换工程师继续。

建议记录：

- 已完成内容。
- 未完成内容。
- 当前阻塞。
- 下一步建议。
- 需要优先验证的风险。
